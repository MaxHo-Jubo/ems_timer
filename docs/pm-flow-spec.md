## 2. 主流程（任務執行循環）

### 待機狀態（IDLE）

- 顯示待機畫面
- 等待操作

### 開始任務（長按主鍵 ≥ 2 秒）

- 建立新任務
- 記錄任務開始時間
- 同步時間
- 開始計時
- 切換至執行模式

### 任務執行中（RUNNING）

- 計時運行
- 定時提醒（6 秒循環）
- 按鍵操作
- 記錄數據
- 顯示資訊

#### 節律提醒循環（每 6 秒，**僅通氣模式啟用**）

1. 提醒開始（蜂鳴 + 震動 + LED 提示）
2. 6 秒倒數（節拍器，誤差 ±50ms）
3. 提醒結束

> 2026-04-23 PM 確認：6 秒 CPR 節拍只在「通氣模式」啟用；切到給藥模式時節拍器停止並重置。
>
> **節拍精度 ±50ms** 為產品可感知規格。韌體實作方式（RTOS timer、計時補正、常數定義）詳見 `pm-dev-spec.md §4.2 節律提醒引擎`（SoT）。

#### 給藥倒數提醒流程（**僅給藥模式啟用**，誤差 ±50ms）

給藥模式（適用 OHCA 等救護流程）進入 RUNNING 後，倒數模組自動啟動，內部分三個階段：

| 階段 | 觸發條件 | 使用者感知行為 |
|------|---------|---------------|
| **COUNTING**（倒數中） | 進入 RUNNING + 給藥模式 | 全螢幕顯示剩餘時間，LED 狀態指示 |
| **WARNING**（預警） | 剩餘 ≤ 1 分鐘 | LED 恆亮 + 閃動，蜂鳴提示加強 |
| **ALARMING**（發報） | 倒數歸零 | 全螢幕閃爍 + 蜂鳴連續發報 + 震動 + 顯示「請給藥」 |

**主鍵短按行為依倒數階段不同**：

| 倒數階段 | 短按主鍵動作 | 說明 |
|---------|-------------|------|
| **ALARMING**（Epi 到時） | 解除發報 + 記錄 Epinephrine + 重置倒數 4 分鐘 | 4 分鐘倒數專為 Epinephrine，ALARMING 表示該給 Epi 了 |
| **COUNTING / WARNING**（倒數中） | 進入**藥物選單**（Amiodarone / TXA / D50 …）→ 選擇藥物 → 記錄該藥物事件 | **不影響 4 分鐘 Epi 倒數**，獨立記錄時間戳 |

> **設計原理**：OHCA 流程中 Epinephrine 每 4 分鐘固定給一次（由倒數模組管控），其他藥物（Amiodarone、TXA、D50 等）依臨床判斷隨時給予，兩者互不干擾。

循環重複直到任務結束（超長按 5s）或暫停（長按 2s）。

> **與狀態機的關係**：COUNTING / WARNING / ALARMING 是給藥倒數模組的**內部階段**，不影響頂層狀態機（IDLE / RUNNING / PAUSE / END）。PAUSE 時倒數暫停、發報停止；恢復 RUNNING 時從剩餘時間繼續。結束任務時一切停止。
>
> 4 分鐘倒數的工程實作常數（`DEFAULT_MED_COUNTDOWN_MS`、`DEFAULT_MED_WARN_1MIN_MS`）詳見 `pm-dev-spec.md §4.2` 與 `firmware/lib/ems_logic/ems_countdown.h`。

### 任務暫停（PAUSE）

- 暫停計時與所有提醒（給藥倒數暫停、通氣節拍停止）
- 若正在給藥發報（ALARMING）→ 立即停止發報
- 保持資料（倒數剩餘時間保留）
- 顯示暫停狀態
- 恢復 RUNNING 時從剩餘時間繼續倒數

#### 判斷

- 是否繼續任務？
  - 是 → 回 RUNNING
  - 否 → 結束任務

### 結束任務（長按主鍵 ≥ 2 秒）

- 停止計時
- 停止提醒
- 儲存任務資料
- 寫入 Flash
- 回到待機

### 背景處理

#### 異常處理

- 按鍵長按處理
- 過早中斷處理
- 錯誤異常
- 儲存異常
- 低電量顯示

#### 資料即時處理

- 按鍵事件
- 時間戳記
- 同步資料
- BLE 傳輸
- 定期寫入 Flash

## 3. 按鍵與事件處理邏輯

> **工程常數對應**：`SHORT_PRESS_MAX_MS` / `LONG_PRESS_MIN_MS` / `VERY_LONG_PRESS_MIN_MS` / `DEBOUNCE_MS` 的具體數值與判定流程（fire-on-release 等）詳見 `pm-dev-spec.md §4.3 按鍵事件處理`（SoT）。本節以使用者可感知的秒數描述行為。

### 主鍵（正下方大鍵）

#### 短按（< 1.5s）

- 記錄事件（依模式，見 §2 給藥倒數流程）
  - 給藥模式 — ALARMING 時：解除發報 + 記錄 Epinephrine + 重置倒數
  - 給藥模式 — 非 ALARMING 時：進入藥物選單 → 選擇藥物 → 記錄事件（不影響 Epi 倒數）
  - 通氣事件（時間記錄 + 提示音）

#### 長按（2s ≤ 按住時間 < 5s）

- 開始任務（IDLE → RUNNING）
- 暫停任務（RUNNING → PAUSE）
- 繼續任務（PAUSE → RUNNING）

#### 超長按（≥ 5s）

- 結束任務（RUNNING 或 PAUSE → END → IDLE）

> 2026-04-23 PM 確認：
> - 長按 2s 用於**進行中的狀態循環**（IDLE↔RUNNING↔PAUSE），降低誤觸門檻
> - 超長按 5s 才會**結束任務**，避免救護現場意外中斷正在進行的任務
> - 兩段式長按設計：2s 時若放開則執行長按行為；繼續按到 5s 則覆寫為結束

### 左側上下鍵（功能切換）

- 短按切換模式（4 種循環，對齊 PM 流程圖）：
  1. 給藥模式（每 4 分鐘提醒）
  2. 通氣模式（每 6 秒節拍）
  3. 自訂模式
  4. 設定模式
- 更新顯示 + 提示音

### 右側鍵（電源）

- 短按（< 1.5s）：螢幕亮 / 滅
- 長按（≥ 2s）：開機 / 關機（含安全判斷）

### 左上轉角鍵（錄音鍵）

- 長按（≥ 2s）：錄音啟停（INMP441 + SD 卡存 WAV，Phase 1.5 模組到貨後啟用）
- 短按：保留（未定義）

> 2026-04-23 PM 確認：
> - 只提供錄音鍵，**不提供靜音鍵**（原靜音鍵條款移除）
> - 錄音採**長按**觸發（對齊 PM 流程圖「左上錄音鍵 長按 開始/停止錄音」）

## 4. 主副機通訊流程（BLE）

> 本節描述**主副機配對與同步流程**。GATT Service 列表、UUID、Characteristic 定義詳見 `pm-dev-spec.md §4.6 BLE 通訊`（SoT）。

### 主副機角色定義（Phase 3）

> 2026-04-23 PM 流程圖確認：主副機為兩台 EMS Timer 硬體，角色綁定如下：

| 角色 | 使用者 | 主要功能 | 節律模組 |
|------|-------|---------|---------|
| **主機**（Central） | 給藥者 | Epinephrine 4 分鐘倒數 + 藥物選單記錄 | 給藥倒數 |
| **副機**（Peripheral） | 給氣者 | 6 秒通氣節拍（由主機遙控啟動） | 通氣節拍器 |

- 主機負責建立任務、管理事件、控制副機啟停
- 副機收到主機指令後啟動通氣節律，持續 6 秒週期直到主機停止
- 單機階段（Phase 2）：同一台裝置透過模式切換兼任兩種角色

### 主機（中心）

- 開機 → 進入主機模式
- 廣播（可被發現）

#### 掃描流程

- 搜尋副機
- 找到設備？
  - 否 → 持續搜尋
  - 是 → 建立連線（配對）

#### 同步內容

- 時間同步
- 事件同步
- 指令同步

#### 連線維持

- 心跳檢測

### 副機（周邊）

- 開機 → 進入副機模式
- 等待連線

#### 收到連線請求？

- 否 → 持續等待
- 是 → 配對連線

#### 同步資料

- 接收主機同步
- 資料接收 / 傳送
- 事件回報

### BLE 傳輸資料內容

- 事件資料（時間戳、類型）
- 裝置狀態（電量、模式）
- 設定參數
- 指令（開始 / 停止 / 切換）

### 連線狀態顯示

- 已連線
- 連線中
- 斷線

## 5. 資料記錄與儲存

### 流程

1. 事件產生（按鍵 / 系統）
2. 加入事件緩衝區（RAM）
3. 暫存（約 5 秒）
4. 寫入 SPI Flash（加密）
5. 標記已寫入
6. 可透過 BLE / Wi-Fi 傳輸至 APP

### 資料結構（單筆事件，概念描述）

每筆事件至少包含：**事件編號 / 時間戳 / 任務經過毫秒 / 事件類型 / 來源（按鍵或系統）/ 裝置 ID / 模式 / 備註 / 同步旗標**。

> 完整 C struct 定義（型別、bytes、欄位順序、`elapsed_ms` 欄位來源）詳見 `pm-dev-spec.md §4.5 資料結構`（SoT）。

### 儲存位置

- SPI Flash（主要）
- SD Card（選配）
