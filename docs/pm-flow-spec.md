## 📘 本文件定位

**本文件管什麼：**
- 使用者可感知的**系統行為**：啟動流程、狀態轉換、按鍵互動、BLE 配對流程、資料流動
- 使用者可感知的**規格參數**：秒數（按鍵 timing）、精度（±50ms）、模式選項
- 硬體層的**產品視角清單**：輸入模組、輸出模組、通訊模組（見 §7）

**本文件不管什麼：**
- 工程實作常數名（如 `LONG_PRESS_MIN_MS`）→ 詳見 `pm-dev-spec.md §4.3`
- C struct 型別與 bytes → 詳見 `pm-dev-spec.md §4.5`
- GATT Service UUID 列表 → 詳見 `pm-dev-spec.md §4.6`
- FreeRTOS Task 分工 / SQL schema / REST API → 詳見 `pm-dev-spec.md` 對應章節

當兩份文件在同一主題上有衝突時，以 **Source of Truth 標註的那一份為準**（每節標示 SoT 歸屬）。

## 1. 系統啟動與初始化

### 電源 ON

- Power Check
- 自我檢測

### 初始化模組

- RTC 時鐘
- 按鍵 / IO
- 顯示驅動
- 震動 / 蜂鳴器
- 藍牙（BLE）模組

- 聲音系統（I2S + SD）
- 記憶體檢查

### 模式判斷

- 是否為主機模式？
  - 是 → 進入主機模式（可連線）
  - 否 → 進入副機模式（等待配對）

### 待機畫面

- 顯示：
  - 時間
  - 電量
  - 模式
  - 連線狀態

## 2. 主流程（任務執行循環）

### 待機狀態（IDLE）

- 顯示待機畫面
- 等待操作

### 開始任務（長按主鍵 ≥ 2 秒）

- 建立新任務
- 記錄任務開始時間
- 同步時間
- 開始計時
- 切換至執行模式

### 任務執行中（RUNNING）

- 計時運行
- 定時提醒（6 秒循環）
- 按鍵操作
- 記錄數據
- 顯示資訊

#### 節律提醒循環（每 6 秒，**僅通氣模式啟用**）

1. 提醒開始（BEEP / 震動）
2. 6 秒倒數（節拍器，誤差 ±50ms）
3. 提醒結束

> 2026-04-23 PM 確認：6 秒 CPR 節拍只在「通氣模式」啟用；切到給藥模式時節拍器停止並重置。
>
> **節拍精度 ±50ms** 為產品可感知規格。韌體實作方式（RTOS timer、計時補正、常數定義）詳見 `pm-dev-spec.md §4.2 節律提醒引擎`（SoT）。

#### 每 4 分鐘檢查（**僅給藥模式啟用**，誤差 ±50ms）

- 是否需要高頻提醒？
  - 是 → 強提醒（聲音 + 震動 + 提示）

> 4 分鐘倒數的工程實作（`DEFAULT_MED_COUNTDOWN_MS`、1 分鐘中途警示、重複提醒週期）詳見 `pm-dev-spec.md §4.2` 與 `firmware/lib/ems_logic/ems_countdown.h`。

### 任務暫停（PAUSE）

- 暫停時間與提醒
- 保持資料
- 顯示暫停狀態

#### 判斷

- 是否繼續任務？
  - 是 → 回 RUNNING
  - 否 → 結束任務

### 結束任務（長按主鍵 ≥ 2 秒）

- 停止計時
- 停止提醒
- 儲存任務資料
- 寫入 Flash
- 回到待機

### 背景處理

#### 異常處理

- 按鍵長按處理
- 過早中斷處理
- 錯誤異常
- 儲存異常
- 低電量顯示

#### 資料即時處理

- 按鍵事件
- 時間戳記
- 同步資料
- BLE 傳輸
- 定期寫入 Flash

## 3. 按鍵與事件處理邏輯

> **工程常數對應**：`SHORT_PRESS_MAX_MS` / `LONG_PRESS_MIN_MS` / `VERY_LONG_PRESS_MIN_MS` / `DEBOUNCE_MS` 的具體數值與判定流程（fire-on-release 等）詳見 `pm-dev-spec.md §4.3 按鍵事件處理`（SoT）。本節以使用者可感知的秒數描述行為。

### 主鍵（正下方大鍵）

#### 短按（< 1.5s）

- 記錄事件（依模式）
  - 給藥事件（時間記錄 + 提示）
  - 通氣事件（時間記錄 + 提示音）

#### 長按（2s ≤ 按住時間 < 5s）

- 開始任務（IDLE → RUNNING）
- 暫停任務（RUNNING → PAUSE）
- 繼續任務（PAUSE → RUNNING）

#### 超長按（≥ 5s）

- 結束任務（RUNNING 或 PAUSE → END → IDLE）

> 2026-04-23 PM 確認：
> - 長按 2s 用於**進行中的狀態循環**（IDLE↔RUNNING↔PAUSE），降低誤觸門檻
> - 超長按 5s 才會**結束任務**，避免救護現場意外中斷正在進行的任務
> - 兩段式長按設計：2s 時若放開則執行長按行為；繼續按到 5s 則覆寫為結束

### 左側上下鍵（功能切換）

- 短按切換模式（4 種循環，對齊 PM 流程圖）：
  1. 給藥模式（每 4 分鐘提醒）
  2. 通氣模式（每 6 秒節拍）
  3. 自訂模式
  4. 設定模式
- 更新顯示 + 提示音

### 右側鍵（電源）

- 短按（< 1.5s）：螢幕亮 / 滅
- 長按（≥ 2s）：開機 / 關機（含安全判斷）

### 左上轉角鍵（錄音鍵）

- 長按（≥ 2s）：錄音啟停（INMP441 + SD 卡存 WAV，Phase 1.5 模組到貨後啟用）
- 短按：保留（未定義）

> 2026-04-23 PM 確認：
> - 只提供錄音鍵，**不提供靜音鍵**（原靜音鍵條款移除）
> - 錄音採**長按**觸發（對齊 PM 流程圖「左上錄音鍵 長按 開始/停止錄音」）

## 4. 主副機通訊流程（BLE）

> 本節描述**主副機配對與同步流程**。GATT Service 列表、UUID、Characteristic 定義詳見 `pm-dev-spec.md §4.6 BLE 通訊`（SoT）。

### 主機（中心）

- 開機 → 進入主機模式
- 廣播（可被發現）

#### 掃描流程

- 搜尋副機
- 找到設備？
  - 否 → 持續搜尋
  - 是 → 建立連線（配對）

#### 同步內容

- 時間同步
- 事件同步
- 指令同步

#### 連線維持

- 心跳檢測

### 副機（周邊）

- 開機 → 進入副機模式
- 等待連線

#### 收到連線請求？

- 否 → 持續等待
- 是 → 配對連線

#### 同步資料

- 接收主機同步
- 資料接收 / 傳送
- 事件回報

### BLE 傳輸資料內容

- 事件資料（時間戳、類型）
- 裝置狀態（電量、模式）
- 設定參數
- 指令（開始 / 停止 / 切換）

### 連線狀態顯示

- 已連線
- 連線中
- 斷線

## 5. 資料記錄與儲存

### 流程

1. 事件產生（按鍵 / 系統）
2. 加入事件緩衝區（RAM）
3. 暫存（約 5 秒）
4. 寫入 SPI Flash（加密）
5. 標記已寫入
6. 可透過 BLE / Wi-Fi 傳輸至 APP

### 資料結構（單筆事件，概念描述）

每筆事件至少包含：**事件編號 / 時間戳 / 任務經過毫秒 / 事件類型 / 來源（按鍵或系統）/ 裝置 ID / 模式 / 備註 / 同步旗標**。

> 完整 C struct 定義（型別、bytes、欄位順序、`elapsed_ms` 欄位來源）詳見 `pm-dev-spec.md §4.5 資料結構`（SoT）。

### 儲存位置

- SPI Flash（主要）
- SD Card（選配）

## 6. APP 資料流程

1. 裝置傳輸（BLE / Wi-Fi）
2. APP 接收
3. 資料解析與驗證
4. 本地儲存（SQLite）
5. 資料呈現（時間軸 / 清單 / 圖表）
6. 匯出（PDF / CSV / EMS 系統）
7. 備份 / 上傳雲端

> App 完整功能模組（裝置管理、即時監控、資料同步、視覺化分析、UI 頁面）詳見 `pm-dev-spec.md §二 App（行動端規格）`（SoT）。
