2 按鍵操作定義
按鍵 觸發方式 動作
主鍵
（正下方大鍵）		 短按 < 1.5s 給藥 ALARMING：解除發報 + 記錄 Epi + 重置倒數
給藥非 ALARMING：進入藥物選單 → 選藥 → 記錄（不影響 Epi 倒數）
通氣：記錄通氣事件
長按 2~5s 狀態循環切換：IDLE → RUNNING → PAUSE → RUNNING → …
超長按 ≥ 5s 結束任務（RUNNING / PAUSE → END → IDLE）
上下鍵
（左側）		 短按 切換模式（給藥 → 通氣 → 自訂 → 設定，4 種循環）
電源鍵
（右側）		 短按 螢幕亮 / 滅
長按 ≥ 2s 開機 / 關機
錄音鍵
（左上轉角）		 長按 ≥ 2s 開始 / 停止錄音（INMP441 + SD 卡，Phase 1.5 啟用）

4 系統啟動流程
1

電源開啟

右側電源鍵長按 ≥ 2 秒開機

2

模組初始化

RTC 時鐘 → 按鍵 IO → OLED 顯示 → 蜂鳴器 → BLE 藍牙 → SPI Flash → I2S 音訊

3

開機確認

蜂鳴器嗶 2 聲 → 記憶體檢查

4

待機畫面（IDLE）

顯示：時間 / 電量 / 當前模式 / BLE 連線狀態 → 等待使用者長按主鍵開始任務

⚡ 超長按 ≥ 5s RUNNING 或 PAUSE → 直接跳到 END → 2 秒後自動回 IDLE
兩段式長按設計：放開才結算（fire-on-release），按到 2s 放開 = 切換狀態；繼續按到 5s = 結束任務。 灰色地帶 1.5s ~ 2s 之間直接忽略，避免誤觸。
6 事件資料結構

每次按鍵觸發記錄一筆事件，每次出勤最多 30 筆

event_id
事件編號
timestamp
絕對時間戳
elapsed_ms
任務經過毫秒
event_type
事件類型
source
來源（按鍵/系統）
device_id
裝置 ID（Phase 3）
mode
記錄時的模式
sync_flag
同步旗標
elapsed_ms：自任務起點扣除所有暫停時間的淨經過毫秒數，App 依此欄位繪製事件時間軸。
